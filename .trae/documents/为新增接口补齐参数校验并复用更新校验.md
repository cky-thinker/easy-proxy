目标：为新增（create）接口加入与更新（update）一致的参数校验，并尽量复用校验逻辑，统一错误返回风格。

涉及服务：
- UserService（createUser / updateUser）
- ProxyClientService（addProxyClient / updateProxyClient）
- ProxyClientRuleService（addProxyClientRule / updateProxyClientRule）
- 其他服务（SysLogService、TrafficStatistic*）当前仅查询/新增日志，不涉及复杂唯一性，暂不调整

复用设计：
- 在各自 Service 内抽取私有校验方法（需要 DAO 参与的唯一性检查留在对应 Service 内），更新与新增统一调用：

1) UserService
- 新增私有方法：
  - validateUniqueUserFields(String username, String mobile, String email, Integer excludeId)
  - validatePassword(String password, boolean required)
  - validateRole(String role)
- createUser：
  - 调用 validatePassword(required=true)
  - 调用 validateRole
  - 调用 validateUniqueUserFields(excludeId=null)
  - 设置默认 enableFlag、createTime
- updateUser（已做字段白名单与唯一性校验）：
  - 用 validateUniqueUserFields(excludeId=user.id) 替换/整合现有重复逻辑

2) ProxyClientService
- 新增私有方法：
  - validateNameUnique(String name, Integer excludeId)
  - validateTokenFormat(String token)
  - validateTokenUnique(String token, Integer excludeId)
- addProxyClient：
  - validateNameUnique(excludeId=null)
  - validateTokenFormat + validateTokenUnique(excludeId=null)
  - 设置默认 status、enableFlag、createTime
- updateProxyClient（已加校验）：
  - 改用上述方法以消除重复

3) ProxyClientRuleService
- 新增私有方法：
  - validateProxyClientExists(Integer proxyClientId)
  - validateRuleNameUnique(Integer proxyClientId, String name, Integer excludeId)
  - validateServerPortRangeAndUnique(Integer port, Integer excludeId)
  - validateClientAddress(String clientAddress)  // host:port 且端口范围 1-65535
  - validateLimits(Integer limitConn, Integer limitRate) // 非负
- addProxyClientRule：
  - validateProxyClientExists
  - validateRuleNameUnique(excludeId=null)
  - validateServerPortRangeAndUnique(excludeId=null)
  - validateClientAddress
  - validateLimits
  - 设置默认 enableFlag、createTime
- updateProxyClientRule（已加校验）：
  - 改用上述方法以消除重复

通用校验工具（可选增强）：
- 新建 util 校验工具类（如 Validators）放置格式类校验：
  - isHex64(token)
  - isValidPort(port)
  - parseAndValidateHostPort(clientAddress)
- Service 内仍保留唯一性校验（依赖对应 DAO）

实现步骤：
1. 在三个 Service 中添加私有校验方法，并将 update* 逻辑切换为调用这些方法（不改变现有行为）。
2. 在 create* 流程前置相同校验，设置默认字段，保持字段白名单原则。
3. 编译验证并进行基本手动验证：
   - 创建/更新用户：用户名、手机号、邮箱重复或格式非法时报错；角色校验。
   - 创建/更新客户端：名称/Token 重复、Token 非 64 位十六进制时报错。
   - 创建/更新规则：端口冲突或范围非法、地址格式非法、同客户端重名、限制为负时报错。

交付结果：
- 代码改动集中于上述三个 Service 文件，统一抛出 RuntimeException，沿用现有 Controller 的错误输出风格。
- 重复逻辑最小化，新增与更新共享相同校验入口。
