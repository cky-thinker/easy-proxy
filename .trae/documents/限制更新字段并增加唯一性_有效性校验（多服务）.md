目标：将更新逻辑统一收敛为“只更新必要字段 + 严格校验”，覆盖以下服务：
- ProxyClientRuleService
- ProxyClientService
- SysLogService（无更新逻辑，仅确认无需调整）
- TrafficStatisticClientReportService（空文件，无需调整）
- TrafficStatisticClientRuleReportService（空文件，无需调整）
- TrafficStatisticService（仅查询，无需调整）
- UserService（已完成：基础字段更新 + 唯一性校验）

具体改动：

1) ProxyClientService.updateProxyClient
- 允许更新字段：name、token、enableFlag
- 不允许更新：status、createTime 等系统字段
- 增加校验：
  - name 若变更：需唯一（排除自身 id）
  - token 若变更：
    - 格式校验：64 位十六进制
    - 唯一（排除自身 id）
- 处理：setUpdateTime，保留既有 setUpdateBy("admin") 风格

2) ProxyClientRuleService.updateProxyClientRule
- 允许更新字段：name、serverPort、clientAddress、limitConn、limitRate、enableFlag
- proxyClientId：不建议更新；若传入并变更，校验目标客户端存在
- 增加校验：
  - serverPort 若变更：在 1-65535，且全局唯一（排除自身 id）避免端口冲突
  - clientAddress：格式 host:port，端口 1-65535（基础校验）
  - name 若变更：在同一 proxyClientId 下唯一（排除自身 id）
  - limitConn、limitRate 若传入：需 >= 0
- 处理：setUpdateTime、setUpdateBy("system")，保持既有风格

3) SysLogService / TrafficStatistic* 服务
- 目前无更新方法或仅查询；不做变更，保持现状

实现方式：
- 复用现有 Dao：通过 selectList 做唯一性检查，ne("id", 当前 id) 排除自身
- 使用简单解析/正则进行 clientAddress 与 token 的格式校验
- 所有校验不通过时抛出 RuntimeException，沿用现有错误处理习惯

验证：
- 编译通过
- 关键路径：用户更新、客户端更新、规则更新；在冲突或格式不合法时返回明确错误信息
